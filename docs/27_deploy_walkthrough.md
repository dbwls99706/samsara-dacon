# 27 — Vercel + Supabase 배포 워크스루 (명령어 복붙)

## Step 1 — Supabase 프로젝트 생성 + SQL 한 번 실행 (15분)

### 1.1 프로젝트 생성

1. https://supabase.com/dashboard 접속
2. 우상단 `New project` 클릭
3. 폼 입력:
   - Name: `samsara-leaderboard`
   - Database password: 자동 생성 (메모만 해둘 것 — 잃어버려도 큰 문제 X, RLS 만 쓸 거라)
   - Region: `Northeast Asia (Seoul)` — DACON 심사자가 한국에서 접속하므로 가장 빠름
   - Plan: **Free**
4. `Create new project` → 약 2분 대기 (프로젝트 부트)

### 1.2 SQL 실행 (테이블 + RLS 정책 한 번에)

좌측 메뉴 → `SQL Editor` → `+ New query` → 아래 SQL 전체 붙여넣고 `Run`.

```sql
-- SAMSARA · 윤회 — 익명 글로벌 리더보드 스키마 + RLS
-- src/services/leaderboard.ts 의 인라인 SQL (라인 189-211) 과 1:1 동일.
-- 한 번만 실행. 재실행해도 idempotent.

create table if not exists leaderboard (
  id bigint primary key generated by default as identity,
  date_seed text not null,
  nickname text not null check (length(nickname) between 1 and 16),
  score bigint not null check (score > 0 and score < 1000000000000000000),
  run_identity text,
  card_ids text[],
  created_at timestamptz default now()
);

create index if not exists idx_leaderboard_date_score
  on leaderboard(date_seed, score desc);

alter table leaderboard enable row level security;

drop policy if exists "lb_read" on leaderboard;
create policy "lb_read" on leaderboard
  for select
  using (true);

drop policy if exists "lb_insert" on leaderboard;
create policy "lb_insert" on leaderboard
  for insert
  with check (
    length(nickname) between 1 and 16
    and score > 0
    and score < 1000000000000000000
  );

-- UPDATE / DELETE 는 정책 없음 = 모든 익명 클라가 차단됨.
```

성공 표시: `Success. No rows returned.` 가 뜨면 OK.

### 1.3 검증

`Table Editor` 좌측 메뉴 → `leaderboard` 테이블이 보이고, 우상단에 🛡 RLS 활성 표시가 있어야 한다.

## Step 2 — Supabase API URL + anon key 가져오기 (3분)

1. 좌측 메뉴 → `Project Settings` → `API`
2. 다음 두 값을 복사 (어딘가 메모장에 임시 저장):

| 라벨 | Supabase 표기 | 사용 위치 |
|---|---|---|
| **Project URL** | `Project URL` (예: `https://abcd1234.supabase.co`) | `VITE_SUPABASE_URL` |
| **anon public key** | `anon` 키 (`eyJhbGc...` 로 시작하는 긴 문자열) | `VITE_SUPABASE_ANON_KEY` |

> ⚠️ `service_role` 키는 **절대 클라에 노출 X**. 반드시 `anon` 키만 사용.
> `anon` 키는 RLS 정책으로 보호되므로 노출 OK (이게 Supabase 의 정상 패턴).

## 트러블슈팅 (자주 막히는 5가지)

### TS-1 · 환경변수가 production 에 안 들어감

**증상**: 로컬 dev 는 리더보드 동작, 배포 URL 은 안 됨. 콘솔에 "[leaderboard] fallback to local" 만 출력.

**원인**: `vercel env add` 시 `production` 스코프를 빼먹음.

**해결**:
```powershell
# 현재 등록된 env 확인
vercel env ls

# production 스코프 누락이면 다시 추가
vercel env add VITE_SUPABASE_URL production
vercel env add VITE_SUPABASE_ANON_KEY production

# env 추가 후에는 반드시 재배포 필요
vercel --prod
```

### TS-2 · Supabase RLS 에 INSERT 거부됨

**증상**: 점수 제출 시 콘솔에 `new row violates row-level security policy` 에러.

**원인**:
- (a) RLS 자체는 켜져 있는데 `lb_insert` 정책이 누락됨
- (b) 닉네임이 17자 이상 / 점수가 음수 / `date_seed` 누락

**해결**:
```sql
-- Supabase SQL Editor 에서
select * from pg_policies where tablename = 'leaderboard';
-- → lb_read, lb_insert 두 정책이 보여야 함

-- 누락이면 Step 1.2 SQL 다시 실행
```

`src/services/leaderboard.ts` 의 `sanitizeNickname` 가 16자로 잘라주므로 (a) 가 아니면 거의 발생 X.

### TS-3 · 도메인 propagate 지연

**증상**: `samsara-dacon.vercel.app` 가 5분 지나도 응답 안 함.

**원인**: Vercel 의 자체 도메인 (`*.vercel.app`) 인데도 가끔 캐시 지연.

**해결**:
```powershell
# 1) DNS 캐시 비우기
ipconfig /flushdns

# 2) 다른 네트워크 (모바일 LTE) 에서 접속 시도
# 3) Vercel 대시보드 → Settings → Domains → 도메인 옆 "Refresh" 클릭
# 4) 그래도 안 되면 https://www.whatsmydns.net 에서 글로벌 propagation 확인
```

보통 30분 이내 해결.

### TS-4 · 캐시 무효화 (배포는 됐는데 옛 버전이 보임)

**증상**: `vercel --prod` 직후 새 코드가 반영 안 됨. 강제 새로고침 (Ctrl+Shift+R) 해도 옛날 화면.

**원인**: `vercel.json` 의 `Cache-Control: max-age=31536000, immutable` 가 `/assets/*` 에만 적용되도록 이미 설정됨. HTML 은 캐시 X. 그래도 보이면 CDN edge 캐시 잔존.

**해결**:
```powershell
# Vercel 대시보드 → 해당 deployment → 우상단 "..." → "Redeploy"
# 옵션: "Use existing Build Cache" 체크 해제 후 redeploy

# 또는 CLI:
vercel --prod --force
```

브라우저 측: F12 → Network 탭 → "Disable cache" 체크 후 새로고침.

### TS-5 · 모바일 첫 로드 느림

**증상**: 데스크톱 LCP 1초인데 모바일은 4초 이상.

**원인 후보**:
- 폰 네트워크 (3G / 약한 LTE) — 본질적 한계
- 폰트 (Pretendard, Galmuri) WOFF2 가 첫 페인트 블록
- Supabase 첫 핸드셰이크 (cold connection)

**해결**:
```html
<!-- index.html <head> 에 폰트 preload 추가 (이미 적용 권장) -->
<link rel="preload" as="font" type="font/woff2" crossorigin
      href="/fonts/Pretendard-Regular.woff2">
<link rel="preload" as="font" type="font/woff2" crossorigin
      href="/fonts/Galmuri11.woff2">

<!-- Supabase 미리 connect (DNS prefetch + preconnect) -->
<link rel="dns-prefetch" href="https://abcd1234.supabase.co">
<link rel="preconnect"  href="https://abcd1234.supabase.co" crossorigin>
```

리더보드는 graceful degradation 으로 첫 화면 블록 X (Step 5 코드 확인 가능). 첫 로드 후 lazy 로딩.
