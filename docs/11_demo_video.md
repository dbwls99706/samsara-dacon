# 11 — 시연 영상 (YouTube) 스크립트

> 시연 영상은 1차 대중 투표 + 2차 심사위원 평가 모두에 영향을 준다. **게임 다음으로 중요한 산출물.**

## 1. 사양

- **플랫폼**: YouTube (공개 또는 일부공개)
- **길이**: **60~90초** (절대 5분 넘기지 말 것)
- **해상도**: 1920×1080 (16:9), 60fps 권장
- **자막**: 한국어 + 영어 동시 (또는 한국어 + 영어 자막 토글)
- **음성**: 나레이션 X, BGM + 효과음만 (가능)

## 2. 필수 포함 항목 (DACON 명시)

✅ 시작 화면
✅ 주요 플레이 구간
✅ 핵심 시스템 (점수, 콤보, 강화 등)
✅ 종료 화면

> "데모 영상만으로 게임 전체 단계를 확인할 수 없는 경우" = **실격**.

## 3. 권장 구성 (60초 버전)

```
00:00–00:05  로고 + 게임 제목 + 한 줄 카피
00:05–00:15  메인 화면 (자동 데모 보여주기) → START 탭
00:15–00:35  핵심 플레이 영상 (콤보, 강화, 점수 상승)
00:35–00:45  결정적 순간 1개 (대박 콤보 / 첫 위기 / 스킬 발동)
00:45–00:55  게임 오버 → 한 판 더 → 메타 진보 (별/코인)
00:55–01:00  로고 + URL + GitHub QR 코드
```

## 4. 권장 구성 (90초 풀 버전)

```
00:00–00:05  로고 + 한 줄 카피
00:05–00:10  배경 (왜 이 게임인가, 1줄 자막)
00:10–00:25  메인 → START → 첫 30초 플레이
00:25–00:50  핵심 시스템 발현 (콤보 / 카드 선택 / 진보)
00:50–01:10  하이라이트 컷 (대박 점수, 위기 극복)
01:10–01:25  반복 플레이 흐름 (게임 오버 → 한 판 더 × 2~3 컷)
01:25–01:30  로고 + URL + 크레딧
```

## 5. 자막 가이드

- 한국어 자막은 큰 글씨 (화면 1/8 크기)
- 영문 자막은 작게 보조 (필요 시)
- 자막은 1줄 max 12자 (한국어), 24자 (영문)
- 자막 출력 시 게임 화면 가리지 않도록 하단 1/5 영역

## 6. 녹화 도구

| 도구 | 플랫폼 | 비고 |
|---|---|---|
| **OBS Studio** | Win / Mac | 무료, 풀 컨트롤 |
| Mac QuickTime | Mac | 가벼움 |
| ScreenToGif | Win | GIF 변환에 좋음 |
| Loom | Web | 빠름 |

> OBS 추천. 1080p 60fps 녹화, MP4 출력.

## 7. 편집 도구

| 도구 | 비고 |
|---|---|
| **DaVinci Resolve** | 무료, 풀기능 |
| CapCut Desktop | 무료, 직관 |
| Premiere Pro | 유료 |
| iMovie | Mac 무료 |

> CapCut 추천 (1인 60~90초 영상엔 충분).

## 8. 컷 편집 원칙

- **평균 컷 길이 2~4초** (긴 컷 X — 시청자 이탈)
- **전환은 컷 (Cut)** 만 사용 (페이드 자제 — 시간 낭비)
- 결정적 순간엔 **0.5x 슬로우 모션** + 줌인 (impactful)
- 음악은 1곡 풀 사용 (CC-BY 또는 자체 생성)
- 효과음 강조 (점수, 콤보 시 SFX 부각)

## 9. 음악 (저작권 안전)

- **YouTube Audio Library** (구글 무료, 라이선스 안전)
- **Free Music Archive** (CC 라이선스)
- **Incompetech (Kevin MacLeod)** — CC-BY (출처 표기)
- **자체 BGM** (게임 BGM 그대로 사용 — 가장 안전)

## 12. 썸네일 (중요)

- **썸네일 = 첫 인상**. 클릭률 결정.
- 메인 게임 화면 + 게임명 + 강한 카피
- 한국어 + 영문 작게
- 색감은 게임 톤 그대로 (일관성)
- 점수나 결정적 순간 캡처 사용

## 14. 녹화 분량 권장

- **풀 플레이 5~10분 분량 녹화** → 거기서 60~90초 컷 편집
- 좋은 컷이 안 나오면 다시 녹화 (편집할 때 후회 X)

## 15. 모바일 시연도 포함

영상 중 5~10초 정도는 모바일 화면도 보여주면 좋음 (반응형 강조).

```
컴포지션 예: 데스크톱 화면 + 우하단 모바일 PIP (Picture-in-Picture)
```
