# 04 — 게임 컨셉 후보 (사용자 선택 대기)

> **결정 기한: 2026-05-08까지 1안으로 좁힌다.** (D-18, 기획서 작성 시간 확보)

각 후보는 다음 항목을 포함:
- 한 줄 정의
- 핵심 루프
- 차별화 포인트
- 화면 스케치 (텍스트)
- 5주 일정 적합성
- 위험 요소

## 후보 A — "한글런 (Hangulun)" — 리듬 + 한글 조합

### 한 줄 정의

> 비트에 맞춰 떨어지는 자모를 조합해 한글을 완성하는 한국형 리듬 게임.

### 코어 컨셉

- 위에서 자음(ㄱ,ㄴ,ㄷ...)과 모음(ㅏ,ㅓ,ㅗ...)이 비트에 맞춰 떨어진다.
- 플레이어는 화면 하단 4개 레인의 **타이밍**에 맞춰 탭/키 입력.
- 자모가 모이면 자동으로 한글 1글자가 조합되고, 완성된 글자가 점수가 된다.
- 같은 단어를 완성하면 콤보 보너스 (예: ㅁ+ㅏ+ㅁ → "맘" → +500, 같은 박자 안에 ㅁㅏ+ㅁㅏ → "마마" → +1500).

### 코어 루프 (60초)

```
0초: 제목 → 곡 자동 시작 (가만히 있어도 BGM은 들림)
3초: 첫 자모 도착 → 탭 (자동 학습)
10초: 첫 한글 1글자 완성 → 큰 점수 팝업
20초: 콤보 시스템 발견 (탭 연속 = 점수 배수)
30초: 첫 단어 완성 ("야!"/"앗!" 같은 1~2 글자 단어)
60초: 곡 후렴 도달 → 자모 빈도 급증 → 첫 진짜 도전
```

### 차별화 포인트

- **한국어 한글 조합 + 리듬**은 거의 사례 없음
- 한글 완성 자체가 시각적 보상 (글자가 화면에 떠오름)
- 단어 완성으로 콤보 디자인이 자연 연결
- 한국 심사위원에게 친숙한 메커닉

### 화면 구성

```
┌─────────────────────────────┐
│ Score 12,840  Combo x32     │
│                             │
│ │ㄱ│  │ㅏ│  │ │  │ㅁ│       │ ← 떨어지는 자모
│ │  │  │ │  │  │ │  │       │
│ │ │  │  │  │ │ │  │ │       │
│ ├──┼──┼──┼──┤              │
│ │ㄴ│ㅏ│ㅁ│ㅗ│              │ ← 4 레인 타이밍 라인
│ └──┴──┴──┴──┘              │
│                             │
│  완성: 남,맘,모              │ ← 최근 완성된 글자
│  콤보: "마마" +1500          │
└─────────────────────────────┘
```

### 5주 일정 적합성

- 채보(노트 패턴) 작성 부담 → 1곡으로 시작, 시간 남으면 2~3곡
- 한글 조합 로직: 자모 결합 → JavaScript에서 `String.fromCharCode(0xAC00 + ...)` 로 가능
- 음악: Free Music Archive / Incompetech 의 CC-BY 음원 (출처 표기 필수)
- 또는 Web Audio API로 8비트 chiptune 자체 생성 (라이선스 X)

### 위험

1. **모바일 4 레인 입력 시 손가락 가림** → 가로 모드 제공 또는 2 레인 단순화 옵션
2. **한글 조합 알고리즘 버그 가능성** → 단어 사전 한정 (50개)
3. **음악 동기화 정확도** → Web Audio Clock 사용 (`audioContext.currentTime`)

### 시연 영상 임팩트: ★★★★★

화면에 한글이 박자에 맞춰 솟아오르는 화면은 매우 매력적.

## 후보 B — "30초 클리커" — 클리커 + 로그라이트

### 한 줄 정의

> 30초마다 끝나는 클리커. 매 판 무작위 빌드를 짜야 한다.

### 코어 컨셉

- 한 판 = **정확히 30초**.
- 30초 동안 화면을 탭/클릭해 자원(코인)을 모은다.
- 시간이 끝나면 코인을 사용해 **3장 중 1장 카드 선택** → 다음 판 영구 강화.
- 다음 판 시작 → 더 강해진 상태로 또 30초 → 또 카드 선택...
- 메타 진보 + 무작위 빌드 = 매 판 다른 전략.

### 코어 루프 (60초 = 2판)

```
0초: 메인 → "START" 단탭 → 첫 30초 시작
0~30초: 클릭/탭 → 숫자가 올라감
30초: 결과 화면 → 3장 카드 등장 → "선택 1장"
       (예: "탭당 +1" / "자동 클릭 1초마다" / "5초 폭발")
35초: 두 번째 30초 시작
```

### 차별화 포인트

- 일반 클리커: 무한 진행, 지루
- 우리: 30초 강제 구간 → "이 판은 무엇을 살릴까" 즉시 선택
- 매 판 카드 풀 무작위 → 같은 게임 다시 안 됨
- 한 세션 10분 = 약 20판 = 매 판 다른 빌드

### 화면 구성

```
┌─────────────────────────────┐
│  💰 1,234   ⏱ 12.5초         │
│                             │
│      ┌─────────┐            │
│      │  [TAP]  │  ← 큰 버튼  │
│      │   💥    │            │
│      └─────────┘            │
│                             │
│  보유 능력:                  │
│  [+1탭] [자동×2] [폭발]      │
│                             │
│  최고: 12,800 (이번: 1,234) │
└─────────────────────────────┘
```

### 5주 일정 적합성

- 클리커는 구현 난이도 가장 낮음
- 카드 효과 시스템: 각 카드 = 객체 1개 (트리거, 효과). 12~20개 카드면 충분
- 메타 통계 저장: localStorage
- 시각 보강: 파티클·셰이크·숫자 팝업 (구현 비용 작음)

### 위험

1. **시각적 단조로움** → 배경·캐릭터·효과로 보강 (도트 캐릭터가 탭마다 점프 등)
2. **카드 밸런싱** → 30초 한정이므로 OP 카드도 큰 문제 X
3. **첫인상 약함** → 메인 화면 = 첫 판 자동 시작으로 보완

### 시연 영상 임팩트: ★★★★

빠른 컷 편집 (탭 → 카드 선택 → 다음 판) 으로 매력 살릴 수 있음.

## 후보 C — "타워머지 (TowerMerge)" — Suika + 디펜스

### 한 줄 정의

> 위에서 떨어지는 머지 결과물이 아래로 흘러 적을 막는 타워가 된다.

### 코어 컨셉

- 화면 위쪽 = Suika 머지 영역. 같은 등급 합치면 큰 등급으로 변환.
- 화면 아래쪽 = 적이 좌→우로 행진.
- 플레이어가 큰 등급 블록을 만들면, 그 블록은 아래쪽 디펜스 라인으로 떨어져 **타워**가 되어 적을 공격.
- 타워는 자기 등급에 따라 공격력·범위가 다름.
- 적이 라인 끝까지 도달 = 라이프 -1, 0이 되면 게임 오버.

### 코어 루프 (60초)

```
0초: 메인 → 자동 데모 → 시작 탭
3초: 위쪽에 첫 블록 떨어짐 → 같은 색이 있으면 자동 머지
10초: 첫 큰 블록 완성 → 아래쪽으로 흘러내려 → 자동 타워화
15초: 첫 적 등장 → 타워가 자동 공격 → 적 처치 보상 코인
30초: 적 빈도 증가 → 타워가 부족
60초: 첫 라이프 손실 OR 첫 위기
```

### 차별화 포인트

- **두 장르 결합** = 시연 영상에서 즉시 "이거 뭐지?"
- 머지의 단조로움 + 디펜스의 긴장감 결합
- 한 화면에 두 게임이 동시 진행 = 멀티태스킹 재미

### 화면 구성

```
┌─────────────────────────────┐
│ 💰 240  ❤❤❤  Wave 3         │
│ ┌───────────────────────┐   │
│ │  ●   ◆  ●●  ●         │   │ ← 머지 영역
│ │ ◆◆ ●  ◆     ●         │   │
│ │ ●  ●● ●●●● ●          │   │
│ │ ● ●● ●●● ●●●●         │   │
│ ├───────────────────────┤   │
│ │  ▲ ▲   ▲ ▲▲           │   │ ← 타워 라인
│ │       ⚔  ⚔  ⚔         │   │
│ │ 👹 👹 👺 👹            │   │ ← 적 행진
│ └───────────────────────┘   │
└─────────────────────────────┘
```

### 5주 일정 적합성

- Matter.js로 머지 물리 1주
- 디펜스 시스템 (적 AI, 타워 공격) 1주
- 결합·밸런싱·UI 1.5주
- Juice + 사운드 0.5주
- = 4주 (1주 여유)

### 위험

1. **두 게임 동시 = 화면이 좁음** → 모바일 세로 화면 절반씩 분할
2. **두 시스템 동시 = 버그 가능성 ×2** → 단순화 (적 종류 3개, 타워 종류 5개로 한정)
3. **머지 룰 + 디펜스 룰 = 학습 곡선 높음** → 첫 30초 강제 튜토리얼

### 시연 영상 임팩트: ★★★★★

두 게임이 한 화면에서 동시 진행되는 모습이 매력적. 30초 컷에 강함.

## 추천

> **추천 1순위: B (30초 클리커)** — 일정·완성도 안전성이 최고. 25점 안정성을 사수하기 좋음.
>
> **추천 2순위: A (한글런)** — 한국 심사위원 정성 평가에서 강함. 단, 음원·채보 부담이 있음.
>
> **추천 3순위: C (타워머지)** — 시연영상 임팩트는 가장 강하지만 두 시스템 결합으로 버그 위험 큼.

## 결정 가이드 (사용자가 고를 때)

1. **안정 → B 선택**: 1인 5주에 1등 가능한 가장 안전한 선택.
2. **참신성 베팅 → A 또는 C**: 정성 평가 1등을 노린다면.
3. **하이브리드**: B로 시작해서 시간이 남으면 A의 "한글" 요소 가미 (ex. 폭발 시 한글 파티클).
