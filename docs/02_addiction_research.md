# 02 — "10분 안에 중독" 게임 심리학 리서치

> 이 게임의 모든 디자인 결정은 다음 원칙들을 통과해야 한다.

## 1. 핵심 프레임워크 — Hooked Model (Nir Eyal)

```
Trigger → Action → Variable Reward → Investment → (다시 Trigger)
```

- **Trigger**: 메인 화면, 푸시(없음 → 시각/청각으로 대체), "한 판 더" 버튼
- **Action**: 가능한 한 적은 클릭으로 다음 판 시작 (1탭)
- **Variable Reward**: **무작위성**이 점수에 가미돼야 도파민 루프가 발생. 결정론적 점수 = 곧 지루함.
- **Investment**: 메타 진보 (영구 강화, 캐릭터 해금, 콜렉션). 사용자가 시간을 "투자"한 만큼 떠나기 어려움.

### 우리 게임에 적용

- **Trigger**: 메인 화면 = 데모 자동 재생 + 큰 START 버튼
- **Action**: 한 번의 입력으로 첫 사이클 시작
- **Variable Reward**: 매 사이클마다 가변 점수 + 가끔 희귀 이벤트 (폭발/콤보/레어 드롭)
- **Investment**: 누적 통계, 영구 해금, 외형 변경 등 **localStorage에 쌓이는 어떤 것**

## 2. Csikszentmihalyi 몰입 (Flow) — 난이도 곡선

```
난이도 ↑
  │     ┌─ 좌절 영역 (너무 어려움)
  │   ╱
  │  Flow channel
  │ ╱
  └─ 지루 영역 (너무 쉬움)
실력 →
```

플레이어 실력은 시간이 지나며 향상. 게임도 같이 어려워져야 흐름 유지.

### 권장 난이도 설계

- **첫 60초**: 무조건 성공. 실패해도 재미 ("의도된 첫 죽음")
- **60~180초**: 첫 도전. 성공률 ~70%
- **180초~**: 점점 어려워지는 동시에, 영구 강화로 난이도 일부 상쇄

### 장치
- 점진 가속 (시간이 지나면 빨라지는 등 자연 난이도 상승)
- 적·아이템·맵의 변종 등장
- 플레이어 강화로 어느 정도 따라올 수 있게 보정

## 3. Juice — 1초 만에 인상 남기는 시각·청각 효과

같은 메커닉이라도 "쥬스"가 풍부하면 압도적으로 재미있다. 1인 개발자가 단기간에 인상 남기는 가장 비용 대비 효과적인 투자.

### 필수 Juice 요소

- **Screen Shake**: 큰 이벤트 (사망, 폭발, 강한 타격) 시 0.1~0.3초
- **Particle Burst**: 점수 획득, 적 처치, 콤보
- **Hit Pause / Time Freeze**: 결정적 순간 0.05~0.1초 정지
- **Easing**: 모든 UI 움직임에 ease-out (선형 이동 금지)
- **Audio Layering**: 베이스 BGM + 이벤트 SFX. 무음 X.
- **숫자 팝업**: 점수 획득 시 위로 떠오르며 사라짐
- **Combo Indicator**: 연쇄 성공 시 화면 가장자리 또는 중앙에 명시적 표시
- **카메라 줌**: 결정적 순간 ±5% 줌

> 참고 자료: Jan Willem Nijman의 "The Art of Screenshake" (GDC 2013) — 1인 개발자에게는 필수 시청. 우리는 그 원칙 그대로 차용한다.

## 4. 손실 회피 (Loss Aversion) 활용

플레이어는 얻는 것보다 잃을 가능성에 두 배 민감하다.

### 적용
- 콤보가 끊기는 시각 효과를 강하게 (아쉬움 = 다음 판 동기)
- "이번 판이 최고 기록 X점 - 1" 표시 → 한 판만 더하면 갱신
- "남은 라이프 1" 표시 → 긴장감

## 5. Goldilocks 세션 길이

심리학·하이퍼캐주얼 데이터에 따르면 가장 잘 박히는 한 세션 길이:

- **30초~3분**: 한 사이클
- **5~10분**: 한 세션 (3~10 사이클)
- 10분 넘기면 피로 누적, 한 번에 끄기 시작

> **우리 게임의 한 사이클 = 30초~2분, 한 세션 = 5~10분 이내로 명시 설계.**

## 6. 시작이 가장 중요하다 — 첫 60초 디자인

심사위원·평가자는 첫 60초를 본다. 이 60초에 모든 것이 들어가야 한다.

| 시점 | 발생 | 목적 |
|---|---|---|
| 0초 | 시작 화면 진입 | 시각적 인상 |
| 1초 | 데모 자동 재생 | "이런 게임" 즉시 이해 |
| 3초 | 첫 입력 가능 | 인터랙션 시작 |
| 5초 | 첫 보상 (점수, 효과) | "재미있다" 첫 신호 |
| 15초 | 첫 도전 / 첫 변화 | 학습 곡선 시작 |
| 30초 | 첫 메타 보상 (콤보, 새 능력 등) | "더 해보고 싶다" |
| 60초 | 첫 사망 OR 도전 격상 | "한 판 더" 트리거 |

**모든 게임 디자인은 이 60초 타임라인을 통과해야 한다.**

## 7. 반복 플레이 유도 — 평가 항목 명시 사항

DACON 평가 항목 "재미 및 몰입도"에 **"반복 플레이 유도"**가 명시. 즉 우리는 단판 게임이 아니라 반복 가능한 구조를 만들어야 한다.

### 반복 유도 메커니즘

1. **로컬 최고점수**: localStorage 저장, 다음 진입 시 표시
2. **글로벌 리더보드**: (선택) 익명 닉네임 기반, 백엔드 1개 (Supabase 익명 / Cloudflare Workers + KV)
3. **메타 진보**: 영구 코인, 영구 해금
4. **데일리 챌린지** (선택): 매일 다른 시드 → 같은 조건에서 점수 비교
5. **콜렉션**: 캐릭터/아이템/스킨 잠금해제
6. **업적 (Achievement)**: 단계별 도전 과제
7. **공유 가능한 결과 화면**: SNS 공유 (선택)

> **최소 1·2·3 + 6은 구현. 4·5·7은 시간 여유에 따라.**

## 8. 모바일 우선 — 입력 비용 최소화

- **버튼 위치**: 화면 하단 1/3 영역 (엄지 영역)
- **버튼 크기**: 최소 44pt × 44pt (Apple HIG)
- **터치 응답**: 50ms 이내 시각 피드백
- **롱탭/제스처 의존 X**: 한 손 엄지 단탭이 기본
- **세로 모드 우선**: 가로 모드는 추가 옵션

## 9. 한국어 텍스트 제로화 원칙

심사위원이 빨리 이해해야 한다. 텍스트는 적을수록 좋다.

- 메뉴 화면 텍스트 < 30자
- 게임 중 UI 텍스트 < 10자 (한 라벨)
- 튜토리얼은 텍스트가 아니라 **첫 게임 자체**

## 10. 끝맺음 디자인 (Endings)

매 사이클의 끝이 다음 사이클의 시작이다.

### 게임 오버 화면 권장 구성

- 큰 점수 (이번 판) ─ 애니메이션 카운트업
- 최고 점수 ─ 갱신 시 폭죽
- "한 판 더" 버튼 ─ 가장 큰 버튼
- 메타 보상 (코인/별 등) ─ 카운트 증가
- 스킨/업적 잠금해제 알림 ─ 가끔 등장
- 공유 버튼 ─ 작게 (필수 아님)

> **중요**: 게임 오버 → "한 판 더" 클릭까지 **2초 이내**에 다시 시작 가능해야 한다.
