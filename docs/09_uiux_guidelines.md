# 09 — UI/UX 가이드라인 (20점 사수)

## 1. 절대 원칙 (위반 시 점수 직격타)

1. **첫 화면에 텍스트 설명 X**. 데모가 자동 재생되어야 한다.
2. **튜토리얼 화면 X**. 게임 자체가 튜토리얼.
3. **2탭 안에 게임 시작**. (메인 → 한 번 탭 → 게임 시작)
4. **게임 오버 → 다시 시작 1탭**. 2초 안에.
5. **모든 입력에 50ms 이내 시각 피드백** (탭 → 즉시 색 변화 등).
6. **모달은 안에서 닫힐 수 있어야** (X 버튼 + ESC + 바깥 탭).

## 2. 컬러 팔레트 원칙

- **5색 이내**로 통일.
- 팔레트 1개 정한 뒤 모든 화면 일관 적용.
- 배경 / 강조 / 위험 / 중립 / 강조2 등 의미 매핑.

### 권장 팔레트 예시 (자유 선택, 컨셉 따라 변경)

#### 도트/네오펑크 풍

```
배경     #0d0221
주조색   #ff2a6d
강조     #05d9e8
중립     #d1f7ff
경고     #f9f871
```

#### 미니멀 모노크롬

```
배경     #f5f5f5
주조색   #1a1a1a
강조     #ff6b35
보조     #4a4a4a

```

#### 한국 전통 단청

```
배경     #FFFFFF
주조색   #C8102E (단청 빨강)
강조     #003876 (단청 파랑)
보조     #FFB81C (단청 노랑)
경고     #00563F (단청 초록)
```

## 3. 타이포그래피

### 한국어

- 본문: **Pretendard** 또는 **Noto Sans KR** (둘 다 OFL)
- 강조 / 도트: **Galmuri** (도트 한글), 눈누 검수 폰트
- 영문 도트: **Press Start 2P** 또는 **VT323**

### 텍스트 크기

| 용도 | 크기 (모바일) |
|---|---|
| 메인 타이틀 | 48~64px |
| 점수 (큰) | 36~48px |
| 점수 (작은) | 18~24px |
| 본문 | 16~18px |
| 보조 | 12~14px |

> **모바일 본문 14px 이하 금지.** 가독성.

## 4. 입력 영역

### 모바일 터치 영역

- **최소 44pt × 44pt** (Apple HIG 기준)
- **버튼 사이 간격 최소 8pt**
- **하단 1/3 영역 우선** (엄지가 닿는 영역)
- **좌우 16pt 안전 마진**

### 데스크톱

- 마우스: 큰 버튼 그대로 OK
- **키보드**: 스페이스(주 입력) / ESC(일시정지) / Enter(확인)

## 5. 마이크로 인터랙션

### 버튼 입력 시 (필수)

```
탭 시작 (pointerdown):
  ├── 색상 0.95 어두워짐 (즉시)
  ├── 크기 0.95배 축소 (즉시)
  └── 효과음 1개 (지연 0)

탭 종료 (pointerup):
  ├── 원래대로 복귀 (200ms ease-out)
  └── 액션 실행
```

### 점수 획득 시

```
숫자 팝업 ("+50")
  ├── 0.0~0.1s: scale 0 → 1.2
  ├── 0.1~0.3s: scale 1.2 → 1.0
  ├── 0.0~0.6s: y -50px (위로 이동)
  └── 0.4~0.6s: opacity 1 → 0
```

### 콤보 시

```
화면 가장자리 깜빡임 0.2s
점수 텍스트 크기 1.2배 0.2s
효과음 (콤보 단계마다 다른 톤)
```

### 실패 / 게임 오버 시

```
화면 셰이크 0.3s
화면 살짝 빨갛게 페이드 0.4s
시간 정지 0.1s
"GAME OVER" 텍스트 페이드인
```

## 6. 이징 (Easing) 가이드

- **선형 이동 금지**.
- 입장: `easeOutCubic` (빠르게 들어와 부드럽게 정착)
- 퇴장: `easeInCubic` (천천히 시작해 빠르게 사라짐)
- 강조 / 충격: `easeOutBack` (살짝 오버슈트)

## 7. 사운드 가이드

- **무음 금지** — 무음은 죽은 게임 같음.
- BGM: 60% 볼륨 기본, 옵션에서 0~100 조절.
- SFX: 80% 기본.
- **로딩 직후엔 무음** (브라우저 자동재생 차단). 첫 입력 시 BGM 시작.
- **음소거 토글** 항상 화면 우상단에 작게 배치.

## 8. 접근성

- 색맹 대응: 색만으로 정보 구분 X (모양·아이콘 병행)
- 키보드만으로도 플레이 가능 (마우스 X 시)
- 배경음 별도 음소거 가능
- (가능하면) 큰 글씨 모드

## 9. 다국어 (한 / 영)

- 토글: 옵션 화면 + 메인 화면 우상단 작게
- 텍스트는 모두 i18n key 경유 (하드코딩 금지)
- 영문은 한국어보다 평균 1.3배 길이를 차지 → 레이아웃 여유 둘 것
- 게임 중 텍스트는 가능한 한 숫자 / 아이콘으로 (텍스트 0)

## 10. 온보딩 (튜토리얼 = 첫 게임)

### 권장 패턴

```
0~3초: 데모 자동 재생 (메인 화면)
3초: START 탭
4초: 게임 시작 — 첫 입력 가능 객체가 화면에 보임
5~10초: 의도된 첫 보상 (탭 → 점수)
10~30초: 더 많은 입력 패턴이 자연스럽게 등장 (적, 폭발, 콤보)
30~60초: 의도된 첫 위기 (게임오버 직전)
60초: 첫 게임 오버 → "한 판 더" 학습
```

### 화면 텍스트로 가르치지 말 것

❌ "ㅁ을 탭해서 점수를 얻으세요"

✅ ㅁ이 화면에 등장 → 사용자가 그냥 탭 → 점수 획득

## 11. 게임 오버 화면 (반복 플레이 결정점)

### 필수 요소

- **이번 점수** (큰 글자 + 카운트업 애니메이션)
- **최고 점수** (작게, 갱신 시 폭죽)
- **한 판 더** (가장 큰 버튼)
- **메타 보상** (코인 +X, 별 +Y 등)
- **공유** (선택)
- **메인으로** (작게)

### 재시작 시간 < 2초

`한 판 더` 탭 → 다음 판 시작까지 2초 이상 걸리면 도파민 끊김.

## 12. 일시정지 화면

- ESC 또는 화면 우측 상단 ⏸ 탭
- 게임이 백그라운드로 돌아가면 자동 일시정지 (`document.visibilitychange`)
- 일시정지 중: 게임 화면 어두워짐 + 가운데 메뉴
- 메뉴: 계속 / 다시 시작 / 메인으로 / 옵션

## 13. 옵션 화면

- 음량 (BGM, SFX 따로)
- 언어 (한 / 영)
- 데이터 초기화 (확인 모달 필수)
- 크레딧 (자산·라이브러리 출처)

## 14. 로딩 화면

- 1초 미만: 스플래시 1개
- 1~3초: 스플래시 + 회전 인디케이터
- 3초 이상: 진행 바 + "콘텐츠 로딩 중..."

> **3초 이상 로딩하지 마라.** 자산 줄이거나 코드 분할.

## 15. 에러 / 빈 상태 처리

- BGM 로딩 실패: 무음으로 계속 진행 + 토스트 1회
- 글로벌 리더보드 실패: 로컬 점수만 표시
- 저장 실패: 메모리에만 저장 + 경고 1회
- **게임 자체가 멈추는 에러는 절대 X**

## 16. 검증 체크리스트 (배포 전)

- [ ] 모든 화면이 320px 너비에서 깨지지 않음
- [ ] 모든 화면이 1920x1080에서 깨지지 않음
- [ ] 가로 ↔ 세로 회전 시 깨지지 않음
- [ ] 다크/라이트 모드 둘 다 깨지지 않음 (강제 한쪽 선택 OK)
- [ ] 폰트가 안 로드된 경우 fallback이 흉하지 않음
- [ ] 한 / 영 토글 시 레이아웃 깨지지 않음
- [ ] 게임 오버 → 한 판 더 → 게임 시작 < 2초
- [ ] 모든 버튼에 hover/active 상태 정의
- [ ] 콘솔 에러 0건
