# 14 — 1차 대중 투표 홍보 전략 (6/8 ~ 6/12)

## 1. 1차 통과 = 상위 10팀 (29팀 중)

### 점수 공식
```
점수 = 제출팀 투표 × 0.6 + 참가팀 투표 × 0.2 + 대중 투표 × 0.2
```

→ **다른 제출팀이 우리 게임을 좋아해야 한다.** (60% 비중)

## 2. 핵심 인사이트

다른 제출팀(=동료 개발자)이 보는 평가 시각:

1. **기술적 인상**: "오, 이거 어떻게 만들었지?"
2. **참신성**: "이 컨셉 새롭다"
3. **완성도**: "버그 없네 / 디테일 좋네"
4. **재미**: 자기도 한 판 하고 싶은 욕구
5. **README/코드**: GitHub 공개니까 코드 품질도 본다

## 3. 동점자 처리 (이를 활용)

> 산출물 최초 업로드 시간이 빠른 팀 우선

→ **5/26 마감 직후 6/4~6/5 사이 산출물 업로드** 가 동점 깨기에 유리.

> 단, 완성도(25점)가 더 중요하므로 **무리한 일찍 제출은 X**.

## 4. 노출 채널

### 공식 (가장 중요)

- **DACON 갤러리** — 우리 게임 페이지가 노출되는 메인 채널
- **DACON 게시판** — 사용자가 직접 글 게재 가능

### 외부 (보조)

- **DACON 디스카드 / 슬랙** (있다면) — 다른 제출팀들이 모여 있을 가능성
- **개발자 커뮤니티**: GeekNews, 디스코드, 개발자 트위터, 페이스북 그룹
- **개인 블로그 / 트위터** — 자연 트래픽
- **GitHub** — README가 잘 쓰여 있으면 starring 통한 추가 노출

## 5. 산출물 업로드 + 즉시 홍보 액션 (6/4)

   - 제목: `SAMSARA · 윤회 — 30초마다 새 운명을 짠다`
   - 본문: 1줄 카피 + 스크린샷 3장 + URL + 시연 영상

3. **트위터/X 1개 (선택)**
   - 게임 GIF + 한 줄 카피 + URL

## 8. 다른 참가자에게 건전한 상호 노출 (윤리 한도 내)

### OK

- DACON 게시판에 자기 작품 소개
- 다른 사람 작품 댓글에 진심 어린 피드백 (그 결과 우리 작품 페이지로 자연 유입)
- 참가자 디스코드/슬랙에서 게임 공유

### NOT OK

- 가짜 계정으로 자기 게임 투표
- 다른 참가자에게 표 거래
- 매크로/봇으로 클릭 수 부풀리기

> "부정행위" 적발 시 실격. 절대 X.

### 좋은 카피의 조건

### 예시
