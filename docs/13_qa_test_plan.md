# 13 — QA 테스트 계획

> 25점(완성도/안정성)을 사수한다. 1인 개발자가 5주 안에 안정성을 잡는 가장 빠른 길은 **체크리스트 기반 수동 QA**.

## 1. 테스트 단계

| 단계 | 시점 | 범위 | 빈도 |
|---|---|---|---|
| Smoke Test | 매 커밋 후 | 코어 루프 작동 확인 | 매번 |
| Functional QA | 알파 (6/2) | 모든 화면·기능 | 1회 |
| Cross-Browser QA | 베타 (6/5) | 4개 브라우저 × 2 플랫폼 | 1회 |
| Performance QA | 베타 (6/5) | FPS, 로딩, 메모리 | 1회 |
| Final QA | 6/7 | 전체 + 시연 영상 검증 | 1회 |

## 2. Smoke Test (매번)

매 코드 변경 후 30초 안에 통과해야 한다.

- [ ] 게임 로딩 (스플래시) 정상
- [ ] 메인 → 게임 시작 가능
- [ ] 게임 1판 정상 종료
- [ ] 게임 오버 → 한 판 더
- [ ] 콘솔 에러 0건

## 3. Functional QA 체크리스트

### 3-1. 메인 화면

- [ ] 로고/타이틀 정상 표시
- [ ] 배경음(BGM) 자동재생 (사용자 첫 입력 후)
- [ ] 데모 자동 재생
- [ ] START 버튼 단일 탭으로 게임 시작
- [ ] 옵션 버튼 클릭 시 옵션 화면 진입
- [ ] 언어 토글 작동
- [ ] 최고점수 표시 (저장된 값)

### 3-2. 게임 플레이

- [ ] 첫 입력에 즉시 반응 (50ms 이내)
- [ ] 점수 카운트 정확
- [ ] 콤보 시스템 정상 (예상대로 증가/끊김)
- [ ] 사운드 정상 (탭, 콤보, 게임오버)
- [ ] 시각 효과 정상 (파티클, 셰이크)
- [ ] 게임 진행 중 화면 회전 → 깨짐 없음
- [ ] 게임 진행 중 탭 비활성 → 자동 일시정지
- [ ] 일시정지 → 정상 복귀
- [ ] 60fps 유지 (FPS 표시 켜고 확인)
- [ ] 1분 이상 플레이 시 메모리 누수 없음

### 3-3. 게임 오버

- [ ] 게임 오버 트리거 정확
- [ ] 점수 카운트업 애니메이션 정상
- [ ] 최고 점수 갱신 시 시각 보상
- [ ] "한 판 더" 단일 탭으로 재시작
- [ ] 메인으로 돌아가기 가능
- [ ] 메타 진보 (코인 등) 누적 정확
- [ ] 공유 버튼 (있다면) 작동

### 3-4. 옵션

- [ ] 음량 조절 즉시 반영
- [ ] 언어 전환 즉시 반영
- [ ] 데이터 초기화 시 확인 모달
- [ ] 데이터 초기화 후 모든 진행 상태 리셋
- [ ] 크레딧 화면 자산 출처 모두 표기

### 3-5. 저장 / 복구

- [ ] 새로고침 후 진행 상태 보존 (또는 의도된 리셋)
- [ ] localStorage 데이터 형태 검증 (개발자 도구)
- [ ] 다른 브라우저에서 시작 → 0부터 시작 (계정 없음)
- [ ] 시크릿 모드에서도 정상 동작

## 4. Cross-Browser QA

| 환경 | 우선순위 | 디바이스 |
|---|---|---|
| iPhone Safari (iOS 16+) | 1 | 실기기 |
| Android Chrome | 1 | 실기기 또는 BrowserStack |
| 데스크톱 Chrome | 1 | Win/Mac |
| 데스크톱 Safari | 2 | Mac |
| 데스크톱 Firefox | 2 | Win/Mac |
| 데스크톱 Edge | 2 | Win |
| Galaxy 인터넷 | 3 | Android |

### 각 환경에서 확인

- [ ] 풀 게임 플레이 1회
- [ ] 사운드 정상
- [ ] 화면 비율 정상 (세로/가로)
- [ ] 폰트 정상 로딩
- [ ] 입력 (탭/클릭/키보드) 정상
- [ ] 60fps 유지

## 5. Performance QA

### Lighthouse 점수 목표 (모바일)

- Performance: 80+
- Accessibility: 90+
- Best Practices: 90+
- SEO: 80+

### 확인 항목

- [ ] LCP < 2.0s
- [ ] FID/INP < 100ms
- [ ] CLS < 0.1
- [ ] 번들 크기 (gzip) < 200KB
- [ ] 5분 플레이 후 메모리 < 100MB
- [ ] 모바일 60fps (저사양에서 30fps 이상)
- [ ] 배터리 과소비 X (5분 플레이 시 -3% 이내)

### 도구

- Chrome DevTools → Lighthouse
- Chrome DevTools → Performance 패널
- Chrome DevTools → Memory 패널 (Heap Snapshot)

## 6. 입력 디바이스 QA

### 모바일

- [ ] 단탭 정확
- [ ] 멀티탭 (있다면) 정확
- [ ] 스와이프 (있다면) 인식
- [ ] 화면 가장자리 시스템 제스처와 충돌 X
- [ ] 가로/세로 회전 인식
- [ ] 키보드 (블루투스) 입력 (있다면)

### 데스크톱

- [ ] 마우스 클릭/드래그
- [ ] 키보드: 스페이스(주 입력)
- [ ] 키보드: ESC(일시정지)
- [ ] 키보드: 방향키 (필요 시)
- [ ] 키보드: Enter(확인)

## 7. 접근성 QA

- [ ] 키보드만으로 전 메뉴 접근 가능
- [ ] 색만으로 정보 전달 X
- [ ] 폰트 렌더링 정상 (시스템 폰트 fallback도)
- [ ] 음소거 가능
- [ ] 화면 흔들림 너무 강하면 옵션 (선택)

## 8. 한 / 영 QA

- [ ] 한국어 모든 텍스트 자연스러움
- [ ] 영어 모든 텍스트 자연스러움
- [ ] 한 ↔ 영 전환 시 레이아웃 깨지지 않음
- [ ] 시스템 언어 한국어인 경우 기본 한국어
- [ ] 시스템 언어 그 외인 경우 기본 영어 (또는 한국어 고정)

## 9. 최종 QA (6/7)

### 통과 조건 (전부 ✅이어야 6/8 제출)

- [ ] 위 모든 체크리스트 통과
- [ ] 콘솔 에러 0건 (전 환경)
- [ ] 시연 영상 60~90초 완성
- [ ] 시연 영상이 실제 플레이와 일치
- [ ] 시연 영상 YouTube 업로드 + URL 확인
- [ ] GitHub 저장소 정상 (README, LICENSE 포함)
- [ ] 배포 URL 정상 동작
- [ ] OG 카드 정상 (https://opengraph.xyz)
- [ ] 모바일 실기기 1회 풀 플레이
- [ ] 데스크톱 1회 풀 플레이
