# 22 — 시연 영상 컷 리스트 (실제 구현 기반 · 2-Act 깊이 노출 재구성)

> docs/11 의 기획을 실제 구현된 화면/시스템 기준으로 갱신한 **72초** 컷 리스트. CapCut 편집 기준.
> 모든 캡처는 **배포 URL** (samsara-dacon.vercel.app) 의 라이브 플레이로 OBS 녹화. 로컬 dev 캡처 금지.
>
> ## 왜 재구성했는가 (전략 근거)
>
> 1차 컷오프 점수 가중치 = **제출팀 60% · 참가팀 20% · 대중 20%**. 즉 영상을 보는 가장 큰 표심은
> **다른 게임 개발자**다. 개발자는 "보스가 있네"가 아니라 "보스 *FSM* 이 있네", "에셋이 0이네",
> "내가 직접 `npm test` 로 검증할 수 있네"에 표를 던진다. 기존 컷 리스트는 평면적 기능 투어라
> 깊이가 영상에 *보이지* 않았다. 이번 재구성의 핵심:
>
> - **Act I (0:00–0:33) — 재미**: 10분 중독 루프를 빠르게. 대중·참가팀 40% 대상.
> - **Act II (0:33–1:08) — 깊이**: 엔지니어링을 *눈에 보이게*. 제출팀 60% 대상.
>   보스 FSM / 게임이론 지형 / 절차적 biome / 윤회 도감 / **조합 수식 머니샷** / **"직접 검증 가능" 개발자 카드**.
> - 모든 수치는 소스 데이터 실측(아래 §0). 영상에 틀린 숫자가 뜨면 개발자 신뢰가 0이 된다.

## 4. 녹화 도구

- **OBS Studio** — 1920×1080, 60fps, NVENC H.264, 16Mbps. 마이크 X (게임 SFX/BGM + 별도 VO 트랙).
- 키캡 표시 OFF. 단, 컷 5(QTE)·컷 9(지형 결정)는 입력이 결정을 만든다는 게 보여야 하므로 **키 입력 오버레이 ON 허용**.
- 컷 14 `npm test` PiP: 별도 터미널 녹화 (배포 URL 무관 — 코드 검증 컷이므로 로컬 OK, 단 실제 통과 화면).

## 5. 편집 (CapCut)

1. OBS 녹화본 import → §1 시간대 split.
2. 자막 트랙(한/영 동시) + **Act II 오버레이 트랙** 별도.
3. Act I↔II 전환(0:33)은 0.15s 글리치/플래시 1회 — "장이 바뀐다" 신호. 그 외 컷 간 0.1s 페이드.
4. 컷 12 수식·컷 14 검증 카드는 가독성 우선 — 최소 1.5s 정지 프레임 확보.
5. 마지막 컷 URL + 로고 정적 2초.
6. Export: 1080p / 60fps / MP4 / 30Mbps.

## 7. 60초 / 90초 변형

- 컷 2·3 통합 (4초): "탭 → 콤보 ×10" 한 호흡.
